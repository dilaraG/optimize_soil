# Платформа калибровки J-функции Леверетта и Брукса–Кори

Веб-приложение для инженера ГМ: загрузка скважинных выгрузок, лаборатория (ККД), подбор J и цепочки БК, сравнение методов.

**Запуск только на локальном компьютере** (браузер + Python). Docker не требуется.

---

## Требования

| Компонент | Версия |
|-----------|--------|
| Python | **3.11** или **3.12** (64-bit), с [python.org](https://www.python.org/downloads/) |
| ОС | Windows 10/11, macOS, Linux |
| Браузер | Chrome, Edge, Firefox (актуальная версия) |
| Диск | ~500 МБ под Python и пакеты; место под Excel/CSV выгрузок — по объёму залежи |

При установке Python на Windows включите пункт **«Add python.exe to PATH»**.

---

## Быстрый старт

### Windows

1. Распакуйте проект в каталог **без кириллицы в пути** (например `C:\GM\calib-app\`).
2. Дважды щёлкните **`run.bat`** или в командной строке из корня проекта:

```bat
run.bat
```

3. Откройте в браузере: **http://localhost:8501**

### macOS / Linux

```bash
chmod +x run.sh
./run.sh
```

Браузер: **http://localhost:8501**

## Установка вручную

Из корня проекта:

```bash
python3.11 -m venv .venv
```

Windows:

```bat
.venv\Scripts\activate
pip install --upgrade pip
pip install -r requirements.txt
streamlit run streamlit_app.py
```

macOS / Linux:

```bash
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
streamlit run streamlit_app.py
```

Скрипты `run.bat` / `run.sh` делают то же самое: при первом запуске создают `.venv` и ставят зависимости.

## Порядок работы в интерфейсе

1. **Лаборатория** — ККД или свои Excel, облако J, данные для БК, границы.
2. **Подбор J функции Леверетта** — скважины, добыча (если есть), калибровка, кнопка **«Запомнить»** снимка.
3. **Брукса–Кори** — те же скважины, калибровка, **«Запомнить»** снимка.
4. **Сравнение методов** — только по сохранённым снимкам J и БК.

Рекомендуемый порядок: лаборатория → J и/или БК (в любом порядке) → сравнение.

---

## Снимки для сравнения J и БК

1. На вкладках **J** и **БК** после расчёта — **«Запомнить результаты по скважинам»** (память текущего сеанса).
2. В разделе **«Сравнение методов»** — блок **«Снимки на диск»**: укажите путь к `.csv` и **«Сохранить в файл»** (полный путь или `data/snapshots/имя.csv`).
3. После перезапуска приложения — **«Загрузить снимки из файла»** (тот же путь или выбор файла в диалоге).

Без сохранения в файл снимки пропадают при закрытии Streamlit.

## Обязательные поля скважинной выгрузки

После загрузки CSV/Excel сопоставьте столбцы в интерфейсе. Минимум:

- `WELL_NAME` — имя скважины  
- код эксплуатационного объекта (`PVTNUM_GDM`)  
- `PORO_GDM`, `PERM_GDM`, `PC`, `SWL_GDM`  
- `Кнг_W` — нефтенасыщенность по истории геомодели  

Без сопоставления обязательных полей калибровка J не запустится (сообщение в интерфейсе).

## Частые проблемы

| Симптом | Что сделать |
|---------|-------------|
| `python` не найден | Установите Python 3.11/3.12, перезапустите терминал; на Windows — `py -3.11` |
| Порт 8501 занят | Закройте другой Streamlit или: `streamlit run streamlit_app.py --server.port 8502` |
| Ошибка `pip install` | Проверьте интернет; корпоративный прокси — настройте pip или установите пакеты offline |
| Не открывается страница | Убедитесь, что в терминале нет ошибки; откройте URL из вывода Streamlit |
| Нет ККД | Положите `ККД_БД.xlsx` в `data/` или загрузите в «Лаборатории» |
| Пустое «Сравнение» | Сохраните снимки J и БК в соответствующих разделах на том же сеансе |
